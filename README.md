# Object Defect Detection & Size Measurement using YOLO and OpenCV

A robust, hybrid computer vision solution that detects physical defects (cracks) on concrete surfaces using YOLOv8 and measures their real-world dimensions (length and width in centimeters) using OpenCV.

---

## Key Features

1. **YOLO Defect Detection**: Runs inference using a custom-trained YOLOv8 model to locate cracks.
2. **OpenCV Oriented Measurement**: Instead of using axis-aligned bounding boxes (which are inaccurate for diagonal defects), the system crops the defect, detects contours, and fits an oriented minimum-area rectangle (`cv2.minAreaRect`) to find the exact physical width and length.

## Core Algorithmic Details

### Scale-Invariant Coin Detection
The system resizes the processing frame to a standardized width of `800 pixels` inside the coin detector before running the Hough Circle Transform. This guarantees that standard coins (e.g., quarters) will fall in a predictable pixel radius range (`40 to 130 pixels`), regardless of whether the original image is 640x640 or 4000x4000. Coordinates are scaled back to the original resolution for rendering.

### OpenCV Contour Defect Fallback
If the YOLO model fails to detect a crack in a highly coarse real-world image, the system falls back to OpenCV contour analysis. It applies thresholding to locate dark shadows, groups them into contours, and filters out texture noise by analyzing the contour aspect ratio (height >> width) and minimum area.
